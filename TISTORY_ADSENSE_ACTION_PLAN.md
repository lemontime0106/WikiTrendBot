# 티스토리 애드센스 콘텐츠 개선 실행 계획

## 목표

`daily-bytes.tistory.com`을 검색 결과와 보도자료를 요약해 자동 발행하는 형태에서 벗어나, 직접 검증한 AI·IT 정보를 제공하는 전문 블로그로 개선한다.

티스토리는 서버와 루트 디렉터리를 직접 관리할 수 없으므로 다음 영역에 집중한다.

- 티스토리 관리자에서 게시글·카테고리·메뉴 정리
- 운영자와 콘텐츠의 신뢰 정보 보강
- 대표 게시글의 직접 경험·검증 자료 추가
- WikiTrendBot의 자동 발행 및 품질 검증 방식 개선

## 프로젝트 반영 상태 (2026-07-20)

WikiTrendBot 쪽에서 실행 가능한 항목은 다음과 같이 반영했다.

- AI·IT 핵심 분야에서 벗어난 키워드와 금융·의료·법률 고위험 조언 기본 차단
- 네이버 검색 링크와 사용자 공식 URL의 실제 본문 수집
- 근거 기반 기획, 글 작성, 독립 편집 검수, 실패 시 1회 자동 수정
- 자동 검색 출처가 있으면 활용하되 링크 개수로 생성을 막지 않고, 본문 충실도·직접 경험 메모를 검사
- 티스토리 RSS 기준 기존 글 중복 검사
- `오늘의 한줄 요약` 등 반복 양산형 문구 차단
- 품질 통과 후 본문이 수정되면 승인 취소 및 재검사 요구
- 품질을 통과한 정확히 같은 본문만 티스토리 업로드 허용

남은 작업은 아래 계획에 적힌 티스토리 관리자 작업과 기존 게시글의 실제 수정·비공개 처리다.

## 1순위: 티스토리 관리자에서 바로 할 일

### 1. 자동 발행 중단

WikiTrendBot의 신규 자동 발행을 일시 중단한다.

현재 상태에서 비슷한 개론형 글을 추가하면 대량 생성 콘텐츠로 보이는 신호가 강해질 수 있다. 기존 콘텐츠 정리와 품질 개선을 마친 뒤 발행을 재개한다.

### 2. 공개 글을 25~35개 정도로 정리

블로그의 핵심 주제를 **AI·IT 도구 실전 활용**으로 좁힌다.

우선 비공개할 범위:

- 여행·카페
- 패션·뷰티·건강
- BTS·스포츠
- 국제정세
- 근거가 부족한 ETF·보험·대출 추천
- 직접 사용하지 않은 제품의 `실사용 후기`
- 사실 오류가 확인된 글
- 다른 게시글과 내용이 크게 겹치는 글

티스토리 `관리자 → 콘텐츠 → 글 관리`에서 공개 상태를 비공개로 변경한다. 바로 삭제하지 않고, 추후 재작성 가능성을 남겨 둔다.

우선 비공개하거나 수정해야 할 대표 글:

| 글 번호 | 게시글 | 조치 사유 |
|---:|---|---|
| 24 | 2026 KBO 올스타전 | 잠실야구장 개장 연도 및 예매 정보 오류 |
| 43 | 여행자보험 비교 | 공식 약관·보험료·비교 기준 부족 |
| 45 | ETF 추천 | 공식 상품 정보·비용·위험 설명 부족 |
| 46 | 아이폰 17 케이스 추천 | 직접 사용 증거 없이 `실사용 후기` 표기 |
| 47 | 부산 감성 카페 추천 | 네 곳이라고 설명하지만 실제로는 세 곳 소개, 방문 증거 부족 |
| 55 | 주담대 금리 계산 | 금융 정보의 공식 근거와 검증 부족 |
| 64 | 원·달러 환율 | 금융·경제 판단 근거와 출처 부족 |
| 84 | 1인 개발자 SaaS 가이드 | 사업자등록·세무 관련 단정적인 설명 |
| 86 | K콘텐츠 수출 | 7,050억 원 예산의 의미를 잘못 해석 |

### 3. 중복 글 통합

티스토리는 일반적인 웹 호스팅처럼 자유로운 301 리디렉션을 설정하기 어려우므로 다음 방식으로 통합한다.

1. 더 좋은 게시글 하나를 대표 글로 선정한다.
2. 다른 글에서 유용한 내용을 대표 글로 옮긴다.
3. 대표 글의 제목·본문·수정일을 최신화한다.
4. 나머지 중복 글은 비공개한다.
5. 관련 내부 링크를 대표 글로 정리한다.

우선 통합 대상:

- 50번 + 82번: 휴먼인더루프
- 38번 + 42번: AI 에이전트 업무 트렌드
- 54번 + 63번: AI 글쓰기 활용법
- 7번·56번·60번: AI 이미지 생성 및 업스케일 관련 글의 역할 재정의

### 4. 카테고리 재편

현재의 `IT / 리뷰 / 금융 / 이슈 / 꿀팁` 구조는 범위가 넓고 사이트의 전문 분야를 설명하기 어렵다.

권장 카테고리:

- AI 도구 실험
- 업무 자동화
- 1인 개발·SaaS
- AI 산업·정책

관리 방법:

- `관리자 → 콘텐츠 → 카테고리 관리`에서 글을 이동한다.
- 글이 1개뿐인 `리뷰` 카테고리는 제거하거나 통합한다.
- 범위가 모호한 `꿀팁`, `이슈`는 구체적인 이름으로 변경한다.
- 공개 글이 없는 카테고리는 삭제한다.

참고: [티스토리 카테고리 관리 안내](https://notice.tistory.com/601)

## 2순위: 신뢰 페이지 만들기

티스토리의 페이지 관리 기능이나 일반 게시글을 이용해 다음 페이지를 만든다.

1. 운영자 소개
2. 문의하기
3. 개인정보처리방침
4. 콘텐츠 작성·검수 및 AI 사용 원칙

페이지를 만든 뒤 `관리자 → 꾸미기 → 메뉴`에서 상단 메뉴에 연결한다. 사용하는 스킨에서 페이지 연결이 불편하면 일반 글로 발행하고 메뉴에 해당 URL을 직접 등록한다.

### 운영자 소개에 포함할 내용

- Daily Byte가 다루는 핵심 주제
- 운영자가 직접 하는 업무와 관심 분야
- AI·IT 관련 경험
- 글을 조사·작성·검증하는 과정
- 오류 수정 요청 방법
- 연락 가능한 이메일 또는 문의 채널

실명을 공개하지 않더라도 필명과 운영 배경을 구체적으로 설명한다.

### AI 사용 원칙 예시

> 본 블로그는 자료 조사와 초안 구성에 생성형 AI를 활용할 수 있습니다. 통계, 가격, 일정, 법률·정책 정보는 공식 자료를 통해 사람이 재확인하며, 직접 사용하지 않은 제품을 실사용한 것처럼 작성하지 않습니다.

### 개인정보처리방침에 포함할 내용

- Google AdSense 사용 사실
- Google 및 제3자의 쿠키 사용 가능성
- IP 주소, 브라우저 정보 등 광고 제공 과정에서 처리될 수 있는 정보
- 댓글·문의 과정에서 수집될 수 있는 정보
- 개인정보 관련 문의 방법
- 정책 시행일과 수정일

참고: [Google 게시자 정책](https://support.google.com/publisherpolicies/answer/10502938?hl=ko)

## 3순위: 대표 글 10~15개 재작성

문장과 글자 수만 늘리지 않는다. 각 글에 직접 검증한 정보와 독자가 재검색하지 않아도 될 정도의 근거를 추가한다.

게시글별 권장 요소:

- 공식 출처 링크
- 직접 촬영한 사진 또는 직접 캡처한 화면
- 테스트 날짜와 제품·모델 버전
- 사용한 프롬프트와 입력 조건
- 비교 기준과 평가 방법
- 결과 표
- 성공 사례와 실패 사례
- 사용 중 발견한 제한 사항
- 운영자의 구체적인 판단
- 글 수정 이력

### AI 모델 비교 글 권장 구조

1. 테스트 목적
2. 테스트한 모델·버전·요금제
3. 동일하게 적용한 프롬프트
4. 코딩·요약·이미지 분석 등 평가 항목
5. 결과 화면과 응답 시간
6. 점수표와 평가 근거
7. 모델별 장점과 실패 사례
8. 어떤 사용자에게 적합한지
9. 공식 가격·모델 문서
10. 테스트의 한계와 수정일

### 작성 시 제거할 표현

근거 없이 다음과 같은 표현을 사용하지 않는다.

- 검색을 통해 확인했다
- 실제 사용자들이 호평한다
- 다수의 후기가 있다
- 실사용 결과
- 검증된 제품
- 반드시 해야 한다
- 가장 안전하다
- 수익이 보장된다

해당 표현이 필요하면 출처, 테스트 과정 또는 직접 경험 증거를 함께 제공한다.

## 4순위: 티스토리 화면과 내비게이션 정리

티스토리 관리자에서 다음 항목을 정리한다.

- 블로그 설명을 AI·IT 실전 블로그에 맞게 변경
- 상단 메뉴에 핵심 카테고리 배치
- 운영자 소개·문의·개인정보처리방침 링크 추가
- 비어 있는 공지사항·최근댓글 모듈 제거
- 필요하지 않다면 방문자 수 모듈 제거
- 글이 없는 카테고리 삭제
- 대표 글 사이에 관련 내부 링크 추가
- 태그를 과도하게 생성하지 않도록 제한
- AI 이미지 한 장과 일반론으로 구성된 고정 형식 중단

스킨 전면 교체는 우선순위가 낮다. 콘텐츠 신뢰도와 메뉴 구조를 먼저 개선한다.

## 애드센스·소유권 확인

티스토리 기본 주소를 사용하는 경우 루트에 `ads.txt` 파일을 직접 업로드하려고 하지 않는다. 티스토리는 수익 메뉴에서 AdSense 계정을 연동하는 방식을 안내한다.

확인 항목:

- `관리자 → 수익 → Google AdSense`에 올바른 계정이 연동되어 있는지 확인
- 공개 HTML의 게시자 ID 끝자리 `2066`이 실제 AdSense 계정과 일치하는지 확인
- AdSense 사이트 목록에 `daily-bytes.tistory.com`이 정확하게 등록되어 있는지 확인
- AdSense에서 사이트 확인을 완료한 뒤 검토 요청
- Search Console에서 `sitemap.xml`이 정상 제출되어 있는지 확인
- 수정한 대표 글의 URL에 색인 생성 요청

현재 공개 소스에는 AdSense 메타 태그와 스크립트가 있으므로 게시자 ID가 맞다면 같은 코드를 스킨 HTML에 중복 삽입하지 않는다.

참고:

- [티스토리 ads.txt 안내](https://notice.tistory.com/2603)
- [AdSense에 새 사이트 추가](https://support.google.com/adsense/answer/12169212?hl=ko)

## WikiTrendBot 개선 작업

### 발행 정책

- 발행 주제를 AI·IT로 제한
- 자동 공개 발행을 중단하고 초안 상태로 저장
- 사람이 승인한 글만 최종 발행
- 하루 단위의 기계적인 발행 주기 제거
- 금융·보험·대출·건강·법률 주제 자동 발행 차단

### 출처 검증

- 공식 출처가 없으면 발행 금지
- 통계·가격·일정·정책·제품 사양에는 출처 URL 필수
- 가능하면 기업 블로그나 뉴스 재인용 대신 정부·기업 공식 문서를 사용
- 출처의 게시일과 글 작성일을 비교
- 출처의 수치를 다른 의미로 확대 해석하지 않는지 검사
- 깨진 링크와 비공식 단축 URL 검사

### 제목·표현 검증

- `실사용`, `직접 비교`, `후기`는 직접 증거가 있을 때만 허용
- `최고`, `필수`, `안전`, `보장` 같은 단정적인 표현 제한
- 제목과 본문에서 소개하는 항목 수가 일치하는지 검사
- 미래 일정과 출시 정보를 공식 발표 없이 확정적으로 작성하지 않도록 제한

### 중복 방지

- 기존 게시글 제목·본문과 의미 유사도 검사
- 유사도가 높으면 신규 발행 대신 기존 글 업데이트 제안
- 동일 키워드의 개론형 글 반복 생성 차단
- 같은 결론·목차·문장 패턴 반복 감지

### 템플릿 개선

- 모든 글에 반복되는 `오늘의 한줄 요약` 제거
- 고정된 `서론 → 목록 → 결론` 구조 반복 완화
- 글의 목적에 맞게 실험기·튜토리얼·비교표·문제 해결 기록 등 형식을 다양화
- AI 이미지 자동 삽입을 기본값으로 사용하지 않기

### 사람 검토 체크리스트

발행 전 다음 항목을 확인한다.

- [ ] 이 글이 기존 글과 실질적으로 다른가?
- [ ] 독자가 이 글을 읽고 다시 검색하지 않아도 되는가?
- [ ] 직접 경험·실험·판단 중 하나 이상이 있는가?
- [ ] 중요한 사실과 수치에 공식 출처가 있는가?
- [ ] 제목이 실제 본문과 일치하는가?
- [ ] 직접 사용하지 않은 것을 실사용했다고 표현하지 않았는가?
- [ ] 금융·건강·법률 등 고위험 조언이 포함되지 않았는가?
- [ ] 이미지와 캡처가 본문을 실제로 뒷받침하는가?
- [ ] AI가 작성한 초안을 사람이 사실 검증했는가?

## 하지 않을 작업

다음 작업은 현재 문제 해결의 우선순위가 아니거나 티스토리 기본 주소에서는 직접 처리하기 어렵다.

- 서버 헤더와 웹서버 설정 변경
- 티스토리 기본 주소 루트에 직접 `ads.txt` 업로드
- 복잡한 301 리디렉션 구축
- `robots.txt` 또는 티스토리 기본 사이트맵 직접 수정
- 승인만을 목적으로 글자 수 늘리기
- 의미 없는 태그 대량 추가
- 신규 글을 더 발행해 게시글 수만 늘리기
- 승인 전 광고 스크립트 중복 삽입
- 콘텐츠 정리 없이 스킨만 교체

핵심은 게시글 수를 늘리는 것이 아니라, 공개 콘텐츠를 줄이고 직접 검증한 AI·IT 대표 글의 품질과 사이트 신뢰도를 높이는 것이다.
